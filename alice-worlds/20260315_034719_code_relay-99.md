Here's a useful git trick: To reset your local branch to match the latest commit from another branch (say, 'main') without losing any changes you've made locally, but keeping all your changes safe for future use, you can run `git reset --soft main`. This command resets your working directory and staging area to match the commits in the specified branch while leaving your index unchanged. It's a great way to save your local changes before pulling updates or merging another branch without overwriting them.
